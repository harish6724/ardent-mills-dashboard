Frontend Architecture
Angular 22 application built entirely from standalone components, Signal-based state management, functional HTTP interceptors, and lazy-loaded routes with feature flag guards.
Component Tree — Animated Signal Flow
HTTP Interceptor Chain
Request Pipeline — left to right on outgoing, right to left on response
1st
authInterceptor
Reads JWT from AuthService signal. Clones request with Authorization: Bearer header.
2nd
loadingInterceptor
Increments global loading signal via queueMicrotask(). Decrements in finalize().
3rd
errorInterceptor
catchError() shows MatSnackBar on 4xx/5xx. Re-throws so store can handle.
API
ASP.NET Core
Request arrives at Kestrel. Returns 200 JSON or error status code.
Signal Store Pattern ngrx/signals
withState()
Initial state slice: items array, total count, loading boolean, error string.
withComputed()
Derived signals: filteredItems, pageCount, hasFailures. Update automatically when base state changes.
withMethods()
Actions: load(), applyFilter(), reset(). Each calls patchState() after the HTTP response.
withHooks()
onInit() triggers initial load. takeUntilDestroyed() ensures subscriptions are cleaned up.
DashboardStore
Uses forkJoin([kpis$, trend$, grainMix$, alerts$]) to load all dashboard data in parallel.

Key Angular 22 Patterns
httpResource
Declarative fetching
httpResource(() => '/api/shipments/' + id()) returns a resource with value(), isLoading(), error() signals. Automatically re-fetches when the input signal changes.
Used in detail views
ShipmentDetailComponent and MillDetailComponent use httpResource bound to route param signals. No manual subscription needed.
OnPush + Signals
Every component
All components use ChangeDetectionStrategy.OnPush. Angular only checks a component when its signal inputs emit a new value — minimal re-render work.
computed() selectors
Derived view-model values (filtered lists, formatted KPIs, chart data arrays) are computed() signals — only recalculate when their dependencies change.
withComponentInputBinding
Route param binding
provideRouter(routes, withComponentInputBinding()) binds route params directly to @Input() signals. No ActivatedRoute.params$ subscription needed.
id = input<string>()
DetailComponents declare id = input() and pass it directly into httpResource(() => url + this.id()). The resource auto-refetches on param change.
