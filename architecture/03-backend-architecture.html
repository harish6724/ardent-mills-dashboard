Backend Architecture
ASP.NET Core 9 Web API with a thin controller pattern, dedicated service layer, EF Core 9 code-first persistence, and SQL Server LocalDB — all wired for Swagger documentation and CORS with the Angular client.
Request Processing Layers
Layer Stack — Animated request traversal
1
Kestrel · HTTP Request
TLS termination, HTTP/2, request parsing. Routes matched by ASP.NET Core routing middleware.
GET /api/shipments?page=1 HTTP/2
2
Middleware Pipeline
CORS → Auth → Routing → Endpoint selection. UseSwagger() added in development only.
app.UseCors() → app.UseAuthentication()
3
Controller — [ApiController]
Thin. Receives bound parameters, delegates to service, returns IActionResult. No business logic here.
public async Task<IActionResult> GetAll([FromQuery] filter)
4
Service Layer
All business logic: filter building, pagination, computed fields. Injected via interface for testability.
IShipmentService → ShipmentService
5
EF Core 9 — AppDbContext
IQueryable composition: Where, OrderBy, Skip/Take. AsNoTracking on all reads. ToListAsync() triggers SQL generation.
db.Shipments.AsNoTracking().Where(…).ToListAsync()
6
SQL Server LocalDB
Parameterized T-SQL generated by EF Core SQL Server provider. Results materialized into entities.
SELECT … FROM Shipments WHERE … OFFSET … ROWS FETCH NEXT 10 ROWS ONLY
7
DTO Projection
Entities mapped to ShipmentDto/MillDto. Decouples API surface from EF model — no lazy loading leaks.
entity → new ShipmentDto(entity.Id, entity.Status, …)
8
200 OK JSON Response
System.Text.Json serializes PagedResult<DTO> with camelCase naming. Content-Type: application/json.
Ok(new PagedResult<ShipmentDto> { Items, Total })

Controller Pattern thin
[ApiController] [Route("api/[controller]")] public class ShipmentsController( IShipmentService svc) : ControllerBase { [HttpGet] public async Task<IActionResult> GetAll( [FromQuery] ShipmentFilter filter) { var result = await svc.GetAllAsync(filter); return Ok(result); } [HttpGet("{id}")] public async Task<IActionResult> GetById(int id) { var dto = await svc.GetByIdAsync(id); return dto is null ? NotFound() : Ok(dto); } }
Service Layer Pattern IQueryable
public class ShipmentService(AppDbContext db) : IShipmentService { public async Task<PagedResult<ShipmentDto>> GetAllAsync(ShipmentFilter f) { var q = db.Shipments.AsNoTracking() .Include(s => s.OriginMill) .AsQueryable(); if (f.Status is not null) q = q.Where(s => s.Status == f.Status); if (f.GrainType is not null) q = q.Where(s => s.GrainType == f.GrainType); var total = await q.CountAsync(); var items = await q .Skip((f.Page-1) * f.PageSize) .Take(f.PageSize) .Select(s => ToDto(s)) .ToListAsync(); return new PagedResult<ShipmentDto> { Items = items, Total = total }; } }
EF Core 9 — AppDbContext
DbContext
DbSet<Mill> Mills
10 seed rows from HasData(). Includes nav props: Shipments, Alerts.
DbSet<Shipment> Shipments
FK to OriginMill. Index on Status, GrainType, DepartureDate.
DbSet<MillAlert> MillAlerts
FK to Mill. Severity enum: Info, Warning, Critical.
DbSet<QualityTest> QualityTests
FK to Mill. Grade enum: No1, No2, Sample. Indexed on Grade, Result.
Key Conventions
AsNoTracking()
All read-only queries skip the EF change tracker — lower memory, faster queries.
Deferred LINQ
Filters are composed before ToListAsync() — one round-trip to the database per request.
Parameterized
EF Core always generates parameterized queries. No risk of SQL injection.
HasData() seeding
Seed data lives in OnModelCreating. Applied via dotnet ef database update.
DTO Strategy
Manual projection
Private static ToDto() method in each service. Simple, no AutoMapper dependency.
Decoupled surface
Adding EF navigation properties never breaks API shape — DTOs are stable contracts.
Immutable records
DTOs declared as C# record types — concise, value-equality, JSON-friendly.
Paged results
PagedResult<T> wraps Items list + Total count for every collection endpoint.
