System Overview
Three-tier web architecture connecting an Angular 22 SPA to ASP.NET Core 9 Web API backed by SQL Server LocalDB. Designed for rapid PoC delivery while mirroring production-ready patterns.
Architecture Diagram — Animated Data Flow
Tier Details
Tier 1
Browser — Angular 22
  • Standalone components, no NgModules
  • Signal-based state (ngrx/signals)
  • ChangeDetectionStrategy.OnPush throughout
  • 3 functional interceptors: auth, loading, error
  • Lazy-loaded feature routes via loadComponent
  • Angular Material M3 with custom theme tokens
  • ng2-charts (Chart.js 4) for KPI visualizations
  • httpResource for declarative data fetching

Tier 2
API — ASP.NET Core 9
  • Minimal controllers with [ApiController] attribute
  • Service layer for all business logic
  • IQueryable filtering — deferred SQL generation
  • DTO projection decouples API surface from EF
  • CORS policy allowing Angular origin
  • Bearer token authentication middleware
  • Swagger / Swashbuckle at /swagger endpoint
  • System.Text.Json with camelCase naming

Tier 3
Data — SQL Server LocalDB
  • Zero-install local database for PoC development
  • Code-first schema via EF Core migrations
  • HasData() seed in OnModelCreating
  • AsNoTracking() for all read-only queries
  • Parameterized queries prevent SQL injection
  • Same EF Core provider as production Azure SQL
  • Navigation properties: Mill has many Shipments
  • Indexed foreign keys on all join columns

Cross-Cutting Concerns
01
Authentication
Bearer JWT token attached by authInterceptor on every outgoing HTTP request. API validates via middleware pipeline.
02
Loading State
Global loading signal managed by loadingInterceptor using queueMicrotask() to avoid change detection timing issues.
03
Error Handling
errorInterceptor catches 4xx/5xx responses and displays Angular Material Snackbar messages. Errors propagate to stores.
04
CORS Policy
ASP.NET Core CORS middleware allows the Angular dev origin (localhost:4200) and permits Authorization headers.
